## Standard Study Workflow

Each study session follows this pattern:

```text
Learn → Practice → Build → Review → Commit
```

### Learn

Understand one new concept.

### Practice

Complete small exercises independently.

### Build

Apply the concept to a useful program or project feature.

### Review

Run the code, fix errors, and write short notes.

### Commit

Save progress using Git and push to GitHub.

---

## Git Workflow

At the end of each study session:

```bash
git status
git add .
git commit -m "Clear commit message"
git push
```
